# Dataset Profile

`datasets/`는 심볼릭 링크이며 실제 경로는 `/home/user1/Storage/seg_dataset`(= `/mnt/data/data/seg_dataset`)입니다.

이 문서는 “로컬에 실제로 있는 데이터셋 자산”을 한눈에 보기 위한 스냅샷 문서입니다(2026-02-19 기준 로컬 스캔).

## 용어(통일)

표에서 “어떤 데이터가 들어있는지”는 아래 용어만 사용합니다(동의어 금지):
- **Image (RGB)**: 3채널 이미지(PNG/JPG).
- **Pixel Mask (Index)**: 라벨(ground truth) 1채널 PNG. 픽셀 값이 클래스 ID(`uint8`/`uint16`)인 마스크.
- **Pixel Mask (RGB Palette)**: 라벨(ground truth) 3채널 PNG. RGB 조합이 클래스(팔레트 CSV 등으로 정의)인 마스크.
- **Annotation (BBox)**: 2D bounding box 주석(JSON 또는 YOLO txt).
- **Annotation (Polygon JSON)**: 폴리곤 기반 주석(JSON, `label` 문자열). 학습용으로는 오프라인 rasterization이 필요.
- **Parquet (Waymo)**: Waymo Perception v2 원천 포맷(디코딩 필요).

## 라벨 정규화 정책(요약)

학습 파이프라인에서는 원천 라벨(JSON/Polygon/Parquet 등)을 직접 파싱하지 않고, 오프라인에서 다음 형태로 정규화하는 것을 기본으로 합니다:
1. 원본 이미지 파일
2. Pixel Mask PNG(채널별, `0/1/255` 또는 index ID; `255: ignore`)

데이터셋별 label-space 차이를 흡수하기 위해 “클래스 룩업 테이블(리맵 테이블)”을 별도로 유지합니다. 자세한 배경/정책은 `docs/DATASET_PROFILE_ANALYSIS.md` 참고.
